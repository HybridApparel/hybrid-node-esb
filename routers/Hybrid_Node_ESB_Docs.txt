
New Order Requests

The requesting system will make a HTTP POST request to the following API endpoint with new order information.

https://tranquil-fortress-90513.herokuapp.com/artGun/orders/new/

The body of the request will be in raw text containing an object with several key value pairs. First is a JSON object with the New Order information paired with a key of "orderJSON".  The two other required key value pairs are "key" which is the Hybrid API key, along with "signature" which is computed with the formula: 

sha1( API Secret + API Key + JSON.stringify(orderJSON) ).  

Please find a sample request below, followed by definitions and notes on the fields.

orderJSON: the JSON object containing the new order information

key: the Hybrid ESP Api Key

signature: the request's authentication signature computed by: sha1( API Secret + API Key + JSON.stringify(orderJSON) )

Order Attributes

type: Always ‘order’ for a new order request; required

time: the time you want associated with the new order request; required

method: always ‘create’ for a new order request; required

mode: production/test flag: auto: represents a production order that will be fulfilled and shipped; debug: represents a test order that will not be fulfilled or shipped; used for testing purposes; required

status: Always ‘In Production’ for a new order request; required

status_code: Always ‘6’ for a new order request; required

xid: the unique order id representing your customer order; required

notes: any notes you want to associated with the order

shiplabel_url: If you are generating your own shipping labels for each order, then specify a URL to your shipping label. Otherwise, leave as an empty string or do not send this property.

pack_url: If you are generating your own packing slips for each order, then specify a URL to your packing list. Otherwise, leave as an empty string or do not send this property.

giftnote_comment: Gift note text that you want to appear on the order packing list. Max length 128 characters.

shipping_carrier: The shipping carrier that will be used to ship the order; ArtGun will provide the values to use for this property based on the shipping methods you want to use; ArtGun provided "MI" to be used. Required.

shipping_priority: The shipping method that will be used to ship the order; ArtGun will provide the carrier code to use based on the shipping methods you want to use; ArtGun provided "4272".  Required.

shipping_account: The account # of the shipping account that the shipment will be billed to; only used when ArtGun should bill shipments to a 3rd party account.

shipping_name: Ship to address contact name. Required. Max length 40 characters.

shipping_address1: ship to address line 1. Required. Max length 40 characters.

shipping_address2: ship to address line 2. Required. Max length 40 characters.

shipping_city: ship to address city. Required. Max length 40 characters.

shipping_state: ship to address state/province/region; certain countries require a 2-character code, please refer to Appendix – State Codes for a list of those countries; for all other countries, either leave blank or specify a name. Required.

shipping_country: ship to address country code; see Appendix - Country Codes for a list of the allowed country codes. Required.

shipping_zipcode: ship to address zip code/postal code. Required for domestic shipments.

shipping_phone: phone number associated with ship to address; if this is not available for your orders, then either provide your customer service number (preferred), or ‘000-000-0000’. Required. Max length 15 characters.

shipping_email: email address associated with ship to address; must be a properly formatted email address. Max length 65 characters.

billing_name: bill to address contact name. Max length 40 characters.

billing_address1: bill to address line 1. Max length 40 characters.

billing_address2: bill to address line 2. Max length 40 characters.

billing_city: bill to address city. Max length 40 characters.

billing_state: bill to address state/province/region; for US, Canada, and Australia, the state code abbreviation should be used (see Appendix – State Codes); for other countries, use a name.

billing_country: bill to address country code; see Appendix - Country Codes for a list of the allowed country codes

billing_zipcode: bill to address zip code/postal code. Max lenght 10 characters.

billing_phone: phone number associated with bill to address; must contain digits only, no special characters. Max length 15 characters.

billing_email: email address associated with bill to address; must be a properly formatted email address. Max length 65 characters

items_quantity: the total number of line items on this order.

items_amount: the total retail price of all line items on this order

items_tax: the total item tax of all items on this order

Items - an array of items

name: the description of the product that you want us to associate with the SKU value you provided. Required. Max length 65 characters.

sku: the product SKU; ArtGun will provide this value for each SKU you have enabled in ArtGun inventory. Required.

quantity: the quantity of this SKU/design combination to ship. Required.

unit_amount: the unit price you invoice your customer for this item.

subtotal_amount: the total line item price you invoice your customer for this item (quantity x unit_amount).

necklabel_binid: the warehouse bin ID location of the neck label to use for this sku/design; required only in cases where different neck labels can be applied to the same SKU/design.

hangtag_binid: the warehouse bin ID location of the neck label to use for this sku/design; required only in cases where different neck labels can be applied to the same SKU/design.

Attributes – an array of Item attributes

type: The type of design or additional feature applied to garment. Required.
     • "DigitalPrint" – a printed design that will go on the front/back of a garment
     • "HeatTransfer" - a neck label
     • "HangTag" - a hang tag

location: Location of design element on garment. Required.
     • For type="DigitalPrint", must be either:
          o "CF" – image will be printed on the front of the garment (Center Front)
          o "FB” - image will be printed on the back of the garment (Full Back)
     • For type="HeatTransfer", must be "FNL"
     • For type="HangTag", must be: "FN"

thumbnail: URL of the thumbnail image associated with the design/neck label/hangtag. Required

preview: URL of the preview image associated with the design/neck label/hangtag. Required.

file_url: URL of the print image associated with the design. Required.

file_extension: file extension of image file - either "jpg" or "png". Required.
